# Ръководство за тестване на Windows Auth Client

## Как работи клиентът

1. **Стартиране**: Клиентът създава сесия с API-то и генерира QR код
2. **QR код**: Съдържа URL: `https://ads-auth.nursanbulgaria.com/auth?token={TOKEN}`
3. **Сканиране**: Мобилното приложение сканира QR кода и отваря URL-а
4. **Аутентикация**: Потребителят въвежда credentials в мобилното приложение
5. **Polling**: Клиентът проверява статуса на сесията всеки 2 секунди
6. **Резултат**: При одобрение/отхвърляне показва съобщение

## Стъпки за тестване

### 1. Компилиране и стартиране

```powershell
# От Visual Studio или:
dotnet build ADS.WindowsAuth.Client
dotnet run --project ADS.WindowsAuth.Client
```

### 2. Проверка на логовете

Логовете се записват в: `{ApplicationDirectory}\LOGS\CURSOR{DateTime}.LOG`

**Какво да проверяваш:**
- `Windows Authentication Client стартиран. API URL: ...`
- `Създадена нова сесия: {SessionId}`
- `QR код URL: https://ads-auth.nursanbulgaria.com/auth?token=...`
- `Проверка на статус за сесия: {SessionId}`
- `Статус на сесия {SessionId}: Pending/Approved/Rejected`

### 3. Тестване на QR кода

**Вариант A: С мобилен телефон**
1. Стартирай клиента
2. Сканирай QR кода с камерата на телефона
3. Отвори URL-а в браузъра
4. Въведи credentials
5. Наблюдавай промяната в статуса

**Вариант B: Ръчно тестване**
1. Стартирай клиента
2. Копирай URL-а от логовете (или от QR кода)
3. Отвори URL-а в браузъра
4. Въведи credentials
5. Наблюдавай промяната в статуса

### 4. Проверка на статуса

**В приложението:**
- `Статус: Очакване на сканиране` (оранжево) - чака сканиране
- `Статус: Одобрено ✓` (зелено) - успешна аутентикация
- `Статус: Отхвърлено ✗` (червено) - отхвърлена аутентикация
- `Статус: Изтекла` (сиво) - сесията е изтекла

**В логовете:**
- `Проверка на статус за сесия: {SessionId}` - всеки 2 секунди
- `Статус на сесия {SessionId}: {Status}` - резултат от проверката

## Често срещани проблеми

### Проблем 1: QR кодът не се генерира

**Причини:**
- API-то не отговаря
- Грешка при създаване на сесия
- Липсва QRCoder библиотека

**Решение:**
- Провери логовете за грешки
- Провери дали API-то работи: `https://ads-auth.nursanbulgaria.com/api/auth/session`
- Провери `appsettings.json` за правилен API URL

### Проблем 2: Статусът не се обновява

**Причини:**
- Timer-ът не работи
- API комуникацията не работи
- Сесията не е намерена

**Решение:**
- Провери логовете за `Проверка на статус`
- Провери дали timer-ът е стартиран
- Провери мрежовата връзка

### Проблем 3: QR кодът не се сканира

**Причини:**
- QR кодът е твърде малък
- QR кодът е размазан
- URL-ът е неправилен

**Решение:**
- Увеличи размера на QR кода (в `Form1.cs` промени `300` на по-голямо число)
- Провери URL-а в логовете
- Тествай с ръчно копиране на URL-а

## Тестване на Lock Screen

1. Стартирай клиента
2. Заключи екрана (Win+L)
3. Трябва да видиш QR код на lock screen
4. Сканирай QR кода
5. Отключи екрана
6. Провери статуса в основната форма

## Debug информация

**Всички важни събития се логват:**
- Стартиране на клиента
- Създаване на сесия
- Генериране на QR код
- Проверка на статус (всеки 2 секунди)
- Промяна на статус
- Грешки

**Логовете са в:** `{ApplicationDirectory}\LOGS\CURSOR{DateTime}.LOG`
